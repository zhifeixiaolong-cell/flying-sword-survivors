## §1 M2 范围

### 1.1 必须实现

- ✅ 单剑模型 (容量 = 1)
- ✅ 笔直去程穿透 + 中等追踪曲线回程
- ✅ 基于距离的速度衰减
- ✅ 速度 × 伤害系数的瞬时伤害模型
- ✅ 锋利度参数影响穿透减速
- ✅ 飞剑速度恒 > 角色速度 (设计不变量 INV-01)
- ✅ 池空时按空格无反应
- ✅ 属性系统双层抽象 (角色御剑能力 × 剑基础属性)
- ✅ 飞剑视觉: 梭形 + 柔光晕 + 渐变拖尾 + 点缀粒子 + 路径残痕
- ✅ 关键瞬间反馈 (出鞘 / 命中 / 远端微颤 / 入鞘)
- ✅ 悬浮剑 + 流动呼吸 + 跟随光标转向
- ✅ 准星 (候选 5, 有剑/无剑两态)
- ✅ 不画指向线

### 1.2 不做 (留给后续里程碑)

- ❌ 主动召回 (M5+ 升级技能)
- ❌ 多剑池 (M5+ 角色升级)
- ❌ 多种飞剑武器 (M5+)
- ❌ 飞剑技能 (消耗多剑触发特殊效果, M5+)
- ❌ 近身攻击 (M3+)
- ❌ 飞剑自转动画 (M5+ 高级剑视觉)
- ❌ 颜色升级 (青/紫/金, 中期及之后)
- ❌ 命中粒子爆裂 (M5+ 戏剧化加强)
- ❌ 屏幕震动 / 屏摇 (M3+ 视效阶段)
- ❌ 音效系统 (单独设计, 本文档不涵盖)
- ❌ 教程关瞄准辅助 (M3+, 但代码留可开关口子)

### 1.3 留口子但 M2 不实现

代码 / 数据结构必须支持, 但 M2 配置为最简形态:

- 飞剑容量上限 (M2 = 1, 代码必须支持 N)
- 锋利度系数 (M2 = 0.7, 代码必须支持每把剑独立配置)
- 速度上限 (M2 无上限, 代码必须支持 buff 系统读写)
- 角色御剑能力倍率 (M2 全 = 1.0, 但 `PlayerStats` 必须存在)
- 飞剑属性是数据 (不写死常量)

## §2 实现路径

### 2.1 实施顺序 (从底层到表面)

**Step 1 — 物理与机制层** (无视觉, 占位图形)

- 飞剑实体类 + 状态机 (`OUTBOUND` / `TURNING` / `RETURNING`)
- 基于距离的速度衰减 — 每帧用 `distance(sword, player)` 计算, **不要**基于时间 (会受帧率影响)
- 笔直去程 + 中等追踪 (180°/s 角速度上限) 回程, 回程目标 = 角色当前位置
- 穿透检测 + 锋利度减速 (穿过敌人后速度 × 0.7)
- 速度 × 伤害系数 = 瞬时伤害
- 资源池 + 空池行为 (容量 = 1, 池空按空格无反应)
- 用占位 (纯色矩形) 显示飞剑, 先把机制跑通

**Step 2 — 属性系统抽象层**

- `SwordConfig` 接口定义基础属性
- `PlayerStats` 持有所有御剑能力倍率
- 实例化飞剑时合并计算: 实际属性 = 基础属性 × 御剑能力倍率
- M2 配置为最简形态 (倍率全 = 1.0), 但接口必须就位
- 倍率叠加方式: 默认乘法 (例如 +20% 初速 → 倍率 × 1.2); 锋利度等通常是替换, 不是叠加

**Step 3 — 视觉层 / 飞剑本体**

- 梭形 50 × 4 px, 核心色 `#F8F8FF`, 柔光晕 `#E0E8FF` (2~4 px 外发光)
- 剑尖永远朝飞行方向; M2 不自转
- 关键瞬间反馈 (出鞘闪 / 命中闪 / 远端微颤 / 入鞘闪) 参数见 §3.2

**Step 4 — 视觉层 / 拖尾系统**

- 渐变剑光痕: 长度 = `currentSpeed / initialSpeed × 100px`, 下限 15, 上限 120
- 点缀粒子: Phaser Particle Emitter, 80~120ms 生成 1 个, 寿命 0.4~0.8s
- 路径残痕: 飞剑每帧位置生成淡痕, alpha 0.15, 寿命 0.3~0.5s

**Step 5 — 视觉层 / 悬浮剑**

- 位置: 角色周围 30px, 在"角色 → 鼠标"方向上; 剑尖朝鼠标, 剑柄朝角色
- 鼠标移动 → 悬浮剑实时跟随转向 (**同一帧内完成, 不能滞后**)
- 流动呼吸: Tween 链, 剑分 6~10 段, 每段 alpha 时间偏移; 周期 1.5s, 线性 (不要 ease), 剑柄 → 剑尖方向
- 暗端 alpha 0.5, 亮端 1.0, 亮区宽度 ≈ 剑身 30%
- 消失/出现: 按空格 → 消失 + 出鞘闪; 入鞘 → 入鞘闪 + 200ms tween 渐入

**Step 6 — UI 层 / 准星**

- `this.input.setDefaultCursor('none')` 隐藏系统光标
- 自定义准星: 中心点 (2×2 实心圆) + 外圈 8 点 (1×1, 半径 6, 等分 45°)
- 有剑/无剑两态: 有剑 = `#F8F8FF` / 全显; 无剑 = `#888888` / 外圈消失
- 状态切换 tween: 有→无 150ms 淡出; 无→有 200ms 渐入
- 准星层级最高, 永远在所有游戏元素之上

### 2.2 调试模式建议

- 按 `` ` `` 键开启 debug HUD, 显示:
  - 飞剑当前速度数值
  - 飞剑当前状态 (`OUTBOUND` / `TURNING` / `RETURNING`)
  - 飞剑距角色距离
  - 池容量状态
  - 命中伤害浮动数字
- 所有数值参数集中在 `Config.ts`, 修改即生效, **不要硬编码**
- 实测建议: 单独写一个 `DojoScene`, 只有 1 个角色 + 静止假人 + 慢速移动假人 + 编队假人, 反复测手感

### 2.3 风险提示

- **追踪强度调参是手感最敏感的部分** — 计划至少 2~3 轮迭代
- **拖尾性能** — Graphics 每帧重绘在 M2 单剑没问题; M5+ 多剑要换 Rope 或 shader
- **悬浮剑指向滞后** — 鼠标输入到悬浮剑转向必须在同一帧内完成, 不能有 lag
- **入鞘判定** — "飞剑接触角色"用碰撞圆判定, 半径 = 角色半径 + 5px, 避免擦边不算

## §4 验收标准

### 4.1 机制验收

- [ ] 同时只能存在一把飞剑
- [ ] 池空时按空格无任何反应 (不积累, 不延迟)
- [ ] 去程是直线, 不偏转
- [ ] 飞剑穿过敌人不停止, 减速
- [ ] 飞剑到达 600px (D_max) 时开始转折
- [ ] 回程追踪角色**当前**位置, 不是发射时的位置
- [ ] 飞剑接触角色身体 = 入鞘, 立刻可以再发
- [ ] 不管角色怎么跑, 飞剑总能追上

### 4.2 手感验收

- [ ] 飞剑出鞘"嗖"地一声有爆发感 (1000 px/s 够快)
- [ ] 单次穿透 3 个以上敌人时, 后面的敌人明显吃低伤害
- [ ] 远端那一瞬间能感受到"凝住"的视觉静止
- [ ] 回程曲线是柔和的, 不机械
- [ ] 玩家有意识地走位时, 能感觉到走位影响了回程伤害分布
- [ ] 池空的 1~2 秒里玩家会本能地走位躲避

### 4.3 视觉验收

- [ ] 飞剑能看出是"光中有剑", 不是纯光也不是纯剑
- [ ] 拖尾长度随速度变化, 速度感强
- [ ] 出鞘 / 命中 / 入鞘的白光闪烁清晰可见但不刺眼
- [ ] 悬浮剑的流动呼吸明显但不抢戏
- [ ] 悬浮剑指向鼠标的反应即时, 不滞后
- [ ] 准星 8 点的有剑/无剑切换清晰

### 4.4 整体验收 (最重要)

- [ ] 玩家玩 5 分钟后, **能不看 HUD 就知道自己有没有剑**
- [ ] 玩家玩 10 分钟后, **会主动用走位调整回程命中目标**
- [ ] 玩家发射飞剑的瞬间, **觉得"爽", 不觉得"软"**
- [ ] 飞剑回到手上的瞬间, **觉得"接住了", 不觉得"消失了"**
- [ ] 整体气质让人想到"御剑", 不是"扔回旋镖", 也不是"打弹幕"

## §5 出问题怎么办 (实测调参速查)

实测时出现以下情况, 按调参方向修正. **调整后, 回填到完整设计文档 §8 数值参数总表**.

| 症状               | 可能的问题                | 调参方向                     |
| ------------------ | ------------------------- | ---------------------------- |
| 飞剑"软", 没爽感   | 初速太低 / 拖尾太短       | V_max ↑ 至 1200, 拖尾系数 ↑  |
| 飞剑"飘", 没份量   | 拖尾太长 / 远端速度太高   | 拖尾上限 ↓, V_min ↓ 至 150   |
| 群伤太强, 玩家乱射 | 锋利度太高                | 锋利度 ↓ 至 0.5              |
| 单体伤害不够       | 锋利度太低 / 伤害系数太低 | 伤害系数 ↑ 至 0.15           |
| 远端凝滞不明显     | V_min 太高                | V_min ↓ 至 150               |
| 回程太机械         | 追踪角速度太高            | 角速度 ↓ 至 120°/s           |
| 回程追不上         | V_min < 角色速度          | 重新校准 V_min, 或角色速度 ↓ |
| 看不清飞剑在哪     | 主体太小 / 颜色对比度低   | 尺寸 ↑ 至 60×5, 或柔光晕加强 |
| 玩家分不清池状态   | 悬浮剑视觉太弱            | 悬浮剑 alpha ↑ / 流动对比 ↑  |
