## 1. 设计哲学

### 1.1 一句话定义

**飞剑系统不是武器系统, 是"修真御剑"的资源管理 + 物理涌现系统;**

### 1.2 三个不变量

任何后续迭代都必须遵守的设计不变量:

- **飞剑速度恒 > 角色速度** (剑永远是主动的, 角色不能跑赢剑);
- **飞剑数量 = 资源池容量** (不是武器堆叠, 是法力资源);
- **机制成长 = 修为成长** (能力解锁通过角色等级提升, 而非装备/科技树);

### 1.3 核心操作循环

> **左手走位 (A/D) + 右手瞄准 (鼠标) + 左手发射 (空格)** 三线分工;

走位有战术意义 (避敌 + 决定回程伤害分布);
瞄准由玩家精确控制 (按下空格瞬间的鼠标方向);
发射是离散决策 (单剑模型, 一发一收);

### 1.4 与同类游戏的位置

| 游戏                     | 自动化程度      | 本作            |
| ------------------------ | --------------- | --------------- |
| 吸血鬼幸存者             | 武器全自动      | ❌ 太被动       |
| Brotato / Magic Survival | 武器半自动      | ❌ 同上         |
| 以撒的结合               | 移动 + 方向射击 | ✅ 操作思路接近 |
| Risk of Rain 2           | 移动 + 鼠标瞄准 | ✅ 控制方案接近 |
| 战神 (利维坦之斧)        | 单武器 + 召回   | ✅ 单剑模型借鉴 |
| 暖雪 (Warm Snow)         | 武器悬浮可视化  | ✅ 持剑表现借鉴 |

## 2. 节奏与数量

### 2.1 节奏档位: A 档 (慢节奏)

每发飞剑都有"份量感", 玩家有时间为每发剑做决策;
节奏由"飞剑往返时间"自然决定, 不是独立冷却计时器;

### 2.2 资源池模型

```
角色 (Player):
  飞剑容量上限: 1  (M2 固定; 后期等级提升)
  当前可用飞剑数: 0 ~ 1
  池中飞剑列表: [Sword] / []
  已派出飞剑列表: [] / [Sword]
```

### 2.3 M2 行为规则

- 容量上限 = 1, 永远只能同时存在一把飞剑;
- 按下空格 → 若池中有剑: 取出 1 把发射; 若池空: **无任何反应**, 不排队不缓冲;
- 飞剑回到角色身边 → 重新入池, 容量恢复;

### 2.4 池空时的设计意图

池空 = 玩家"无武器"状态, 持续约 1.5~2.5 秒 (飞剑往返时间);
这是被有意设计的**张力时刻** — "剑出鞘, 人无防", 玩家必须靠走位活下来;
M3+ 将通过"近身攻击"补位 (M2 不实现);

## 3. 轨迹与物理

### 3.1 飞行阶段划分

一次完整的飞剑生命周期分为 5 个阶段:

```
[出鞘] → [去程飞行] → [远端转折] → [回程飞行] → [入鞘]
```

### 3.2 去程: 笔直穿透

- **方向**: 按下空格瞬间, 角色中心 → 鼠标位置的方向向量, 单位化后作为飞行方向;
- **轨迹**: 直线, 无偏转;
- **穿透**: 穿过路径上所有敌人, 无次数上限;
  - 穿透不停止 (不像穿透-3 的"穿够 N 个就停");
  - 穿透代价由"速度损失"自然涌现, 不另设衰减率;
- **终止条件**: 飞剑距离角色发射点达到"最大距离"时, 进入"远端转折"阶段;

### 3.3 远端转折

- 到达最大距离时, 速度自然衰减到最低值 (≈ 200 px/s);
- 视觉上呈现"凝在远端"的瞬间, 拖尾收缩到极短, 剑身微颤;
- 转折瞬间, 速度向量从"远离角色"翻转为"靠近角色", 进入回程;

### 3.4 回程: 中等追踪曲线

- **机制**: Homing 制导 — 每帧根据"飞剑当前位置 → 角色当前位置"的向量, 修正飞剑速度方向;
- **追踪强度**: 中等 (角速度上限约 **180°/秒**);
  - 玩家站着不动: 回程几乎是直线返程;
  - 玩家快速走位: 回程形成柔和 C 形弧, 切过角色当前路径;
- **气质**: "仿佛在一直寻找主人的位置", 不是机械制导, 是有灵性的追踪;

### 3.5 入鞘

- 飞剑接触角色身体时:
  - 飞剑销毁;
  - 池容量 +1;
  - 触发"入鞘闪"视觉反馈;
  - 悬浮剑重新出现在角色身边;

### 3.6 速度规则: 基于距离衰减

**核心公式 (设计层面, 非最终实现)**:

```
速度 V(d) = V_min + (V_max - V_min) × f(distanceRatio)
其中:
  d = 飞剑距角色的当前距离
  distanceRatio = d / D_max
  f(x) = 1 - x  (M2 起步用线性, 后期可调为非线性曲线)
```

**通俗描述**:

- 飞剑离角色越**远** → 速度越**慢**;
- 飞剑离角色越**近** → 速度越**快**;
- 穿过敌人 → 速度乘以"锋利度系数" (例如 0.7);

**关键属性**:

| 属性             | 起点数值  | 说明                   |
| ---------------- | --------- | ---------------------- |
| V_max (初速)     | 1200 px/s | 出鞘瞬间速度 (整体 +20%) |
| V_min (远端速度) | 360 px/s  | 远端转折时速度 (200 → 300 → 360, 整体 +20%) |
| D_max (最大距离) | 600 px    | 距角色最远飞行距离     |
| 速度下限         | 120 px/s  | 物理兜底, 防止剑卡死 (整体 +20%) |
| 速度上限         | 不写死    | 留给 M5+ buff/技能突破 |

### 3.7 速度不变量: 飞剑速度恒 > 角色速度

这是**强约束**, 不是临时兜底;
设计意图: 飞剑永远是主动的, 角色不能靠跑赢飞剑来"丢弃"它;
实现层面: 角色移动速度上限设定时, 必须保证 < V_min (200 px/s), 包括所有 buff 状态;

### 3.8 实现提示

- 用 `update()` 循环每帧计算 `distance(sword, player)`, 据此设速度;
- 不要用基于时间的衰减 (例如"每秒降 X"), 因为这会受帧率影响;
- 基于距离衰减的好处: **物理可预测, 调参所见即所得**;
- 飞剑状态机建议三个状态: `OUTBOUND` / `TURNING` / `RETURNING`;

## 4. 伤害模型

### 4.1 核心公式

```
瞬时伤害 = 当前速度 × 伤害系数
```

每次飞剑与敌人发生碰撞, 计算碰撞瞬间的飞剑速度, 乘以伤害系数, 即为该次命中的伤害;

### 4.2 起点数值

| 属性               | 数值 | 说明                                     |
| ------------------ | ---- | ---------------------------------------- |
| 伤害系数           | 0.1  | 速度 1000 → 伤害 100; 速度 200 → 伤害 20 |
| 锋利度 (M2 基础剑) | 0.7  | 穿过一个敌人后保留 70% 速度              |

### 4.3 涌现效果

这套模型不需要单独设计衰减规则, 以下效果自动涌现:

**涌现 1: 去程"前贵后贱"**

- 第 1 个敌人吃满伤害 (≈100);
- 第 2 个吃 70% 速度的伤害 (≈70);
- 第 3 个吃 49% (≈49);
- 第 4 个吃 34% (≈34);
- 鼓励玩家**优先打击最危险的敌人** (通常在前排), 而不是朝人多方向乱射;

**涌现 2: 回程"由远及近递增"**

- 回程从最远点 (200 px/s) 开始, 远端敌人吃 ≈20 伤害;
- 飞剑加速回来, 越靠近角色速度越快;
- 接近角色的敌人吃 ≈100 伤害;
- 对塔防场景天然合身: **离城墙近的敌人最危险, 飞剑回程刚好惩罚它们**;

**涌现 3: "凝在远端"的真实感**

- 远端转折瞬间速度最低, 视觉上像剑"凝住";
- 此处不是手动写"悬停 0.3 秒", 而是物理涌现;
- 剑势衰减 = 武学/修真直觉;

**涌现 4: 武器差异化只需一个参数**

- 高锋利度剑 (例如 0.9): 穿透损失小, 群伤强;
- 低锋利度剑 (例如 0.4): 穿透损失大, 单体精打;
- M5+ 多武器系统只需调整 `sharpness` 字段;

### 4.4 走位的伤害含义

由于回程是"中等追踪 + 朝角色当前位置", **玩家走位直接决定回程伤害分布**:

- 站着不动: 回程基本是去程的镜像;
- 向左移动: 回程曲线偏左, 切过角色左侧的敌人;
- 向右移动: 回程曲线偏右, 切过右侧敌人;

走位的战术意义被推到极致, 这是单剑模型 + 双程伤害组合的核心乐趣;

## 5. 属性系统抽象

### 5.1 双层属性模型

```
角色 (Player):
  御剑能力 = {
    初速倍率: 1.0,
    最大距离倍率: 1.0,
    锋利度倍率: 1.0,
    伤害系数倍率: 1.0,
    回程速度倍率: 1.0,
    ...
  }

飞剑 (Sword):
  基础属性 = {
    initialSpeed: 1000,
    maxDistance: 600,
    minSpeed: 200,
    sharpness: 0.7,
    damageCoefficient: 0.1,
    ...
  }

发射时:
  实际属性 = 基础属性 × 御剑能力倍率
  用实际属性创建飞剑实例
```

### 5.2 M2 阶段简化

M2 阶段:

- 角色御剑能力**全部 = 1.0** (无加成);
- 飞剑只有一种 (基础剑);
- 实际属性 = 基础属性;

但**框架必须建立**, 否则 M5+ 加角色升级、多种飞剑时需要大幅重构;

### 5.3 关键原则: 属性是数据, 不是常量

**反例 (M2 阶段不要这么做)**:

```
const SWORD_SPEED = 1000;  // ❌ 写死
sword.velocity = SWORD_SPEED;
```

**正确做法**:

```
sword.velocity = player.getEffectiveSwordSpeed();  // ✅ 经过角色×剑属性计算
```

理由: 你已经明确后期会有 buff 系统、装备系统、技能系统, 任何"写死"的常量都会成为后期重构的债;

### 5.4 实现提示

- 建议 `SwordConfig` 接口定义基础属性, 实例化时合并角色 modifier;
- `PlayerStats` 持有所有倍率, 由等级 / 技能 / buff 系统读写;
- 倍率计算: 乘法叠加 (例如 buff 给 +20% 初速 → 倍率 × 1.2);
- 加法 / 乘法 / 替换的选择由具体属性决定 (例如锋利度通常是替换, 不是叠加);

## 6. 视觉设计

### 6.1 整体风格

**流派 2 (写意剑光) + 流派 3 (戏剧化关键瞬间) 结合**:

- 写意: 主体是流动的剑光, 非实物剑;
- 戏剧化: 关键瞬间 (出鞘 / 命中 / 转折 / 入鞘) 有强反馈;

**M2 投入档位**: 中等 (β 级别) — 主体静态 + 关键瞬间微反馈, 不做复杂动画;

**叙事曲线** (颜色随等级演进):

- M2 基础剑: 素白 (新手剑修, 御剑初成);
- 中期: 青 (修为有成);
- 后期: 紫 / 金 (高阶剑诀);
- 终局: 五色 / 流光 (剑神 / 魔头);

颜色本身成为进度反馈, M2 的素白是叙事起点;

### 6.2 飞剑本体

| 属性       | 规范                            | 说明                                |
| ---------- | ------------------------------- | ----------------------------------- |
| 形状       | 梭形                            | 两端尖, 中间略宽, 像细长柳叶 / 流星 |
| 尺寸       | 50 × 4 px                       | 长宽比约 12:1, "比较细的剑光"       |
| 核心颜色   | #F8F8FF                         | 接近纯白, 微弱冷调                  |
| 柔光晕颜色 | #E0E8FF                         | 略带冷蓝, 包裹在核心外 2~4px        |
| 柔光实现   | Glow filter / 多层 alpha sprite | 视性能选择                          |
| 朝向       | 剑尖永远朝飞行方向              | M2 不自转                           |
| 自转       | ❌ 无                           | M2 朴素表现, 自转留给 M5+ 高级剑    |

### 6.3 拖尾与粒子

**拖尾形态**: 渐变剑光痕 (方案 B);

| 属性            | 规范                                  |
| --------------- | ------------------------------------- |
| 起点 (紧贴剑尾) | 颜色 #F8F8FF, alpha 1.0, 宽度 4px     |
| 终点 (拖尾末端) | 完全透明, 宽度 1px                    |
| 长度公式        | `currentSpeed / initialSpeed × 100px` |
| 长度下限        | 15 px (兜底)                          |
| 长度上限        | 120 px (防止速度突破时拖尾失控)       |

**拖尾长度耦合速度**:

- 出鞘 (1000 px/s) → 拖尾 100px;
- 远端 (200 px/s) → 拖尾 20px (会被下限锁到 15px);
- 回程加速 → 拖尾重新生长;
- 玩家通过拖尾长度, 视觉上感知到飞剑的伤害状态;

**点缀粒子**:

| 属性     | 规范                                 |
| -------- | ------------------------------------ |
| 生成速率 | 每 80~120ms 生成 1 个 (有随机偏移)   |
| 生成位置 | 拖尾中段附近, 随机偏移 ±5px          |
| 尺寸     | 1~2 px (随机)                        |
| 颜色     | #F8F8FF (与剑光同色), 初始 alpha 0.6 |
| 运动     | 慢速漂移 ≈ 30 px/s, 方向随机         |
| 寿命     | 0.4~0.8 秒 (随机)                    |
| 消散方式 | alpha 线性降到 0                     |

**飞行路径残痕** (消散-2):

| 属性     | 规范                             |
| -------- | -------------------------------- |
| 触发     | 飞剑飞过的每个位置都生成一段淡痕 |
| 颜色     | 同剑光, 初始 alpha 0.15 (极淡)   |
| 寿命     | 0.3~0.5 秒                       |
| 消散方式 | alpha 线性降到 0                 |

整体气质: "剑过留痕, 痕迹很快消散" — 存在但不抢戏;

### 6.4 关键瞬间反馈 (β 级别)

**出鞘闪**:

- 触发: 玩家按空格, 飞剑出现;
- 视觉: 角色身上短促白光 (alpha 0 → 1 → 0, 总时长 150ms);
- 颜色: #FFFFFF;
- 范围: 直径约 30px 的圆形光晕;

**命中闪**:

- 触发: 飞剑碰撞敌人;
- 视觉: 剑身整体 alpha 瞬间拉高到 1.2 (溢出泛白), 持续 80ms 后恢复;
- 同时拖尾"瞬间变长" 20%, 80ms 后恢复 — 制造"剑卡了一下"的打击感;

**远端微颤**:

- 触发: 飞剑到达最大距离, 进入转折状态;
- 视觉: 剑身位置抖动 ±1px, 频率 30Hz, 持续 100ms;
- 同时拖尾收缩到下限 (15px);

**入鞘闪**:

- 触发: 飞剑接触角色, 销毁前;
- 视觉: 与出鞘闪相同, 150ms 白光;
- 然后悬浮剑重新出现 (200ms tween 渐入);

### 6.5 悬浮剑 (待机状态)

**位置**:

- 飞剑悬浮在**角色周围**, 距离角色中心 **30px**;
- 方向: 始终在"角色 → 鼠标"方向上;
  - 例: 角色在 (640, 600), 鼠标在 (800, 200) → 悬浮剑在角色到鼠标方向 30px 处;
- 剑尖朝鼠标方向, 剑柄朝角色;
- **玩家移动鼠标 → 悬浮剑实时跟随转向**;

**双重功能**:

1. **池状态可视化** — 有剑时显示, 池空 (剑在外飞行时) 不显示;
2. **瞄准反馈** — 剑的朝向就是按下空格后飞剑的飞行方向, 替代传统指向线;

**待机表现: 流动呼吸**:

| 属性       | 规范                                                          |
| ---------- | ------------------------------------------------------------- |
| 实现方式   | Phaser Tween 链 (做法 D), 剑分成 6~10 段, 每段 alpha 时间偏移 |
| 流动方向   | 剑柄 → 剑尖 ("灵气从主人流向远端")                            |
| 周期       | 1.5 秒                                                        |
| 缓动函数   | 线性 (匀速, 不要 ease)                                        |
| 亮区宽度   | 约剑身的 30% (≈ 15px)                                         |
| 暗端 alpha | 0.5 (即使最暗也保持可见)                                      |
| 亮端 alpha | 1.0                                                           |
| 循环       | 无停顿, 流到剑尖立即从剑柄重新开始                            |

**消失/出现**:

- 玩家按空格 → 悬浮剑消失 + 出鞘闪 + 飞剑实体生成;
- 飞剑入鞘 → 入鞘闪 + 悬浮剑重新出现 (200ms tween 渐入);

### 6.6 实现提示

- 飞剑本体可用单个 sprite + glow shader, 或 Graphics 多层填充;
- 拖尾建议用 Phaser 的 Rope 或自定义 Graphics 每帧重绘;
- 粒子用 Phaser Particle Emitter 即可;
- 悬浮剑流动呼吸用 Tween 链最简单, 后期可换成 shader UV 偏移;
- 所有视觉元素的颜色、尺寸、时长建议放在统一的 `VisualConfig.ts` 中, 方便调;

### 7.1 准星: 候选 5 (同心圆 + 八点星点)

**设计概念**: "八方灵气汇聚于一点" — 道家"八方"概念, 灵气从八方汇聚到中心点, 这一点就是飞剑会到的位置;

**无剑状态**:

- 有剑 → 无剑: 外圈 8 点 0.15 秒 tween 淡出, 中心点同时变灰;

### 7.2 不画指向线

明确**不实现**:

- ❌ 角色到鼠标的指向线;
- ❌ 角色身上的方向箭头;
- ❌ 任何 HUD 形式的瞄准辅助;

- 隐藏系统默认箭头光标;
- 用候选 5 准星替代;
- 准星跟随鼠标实时移动, 无延迟;

- Phaser 隐藏系统光标: `this.input.setDefaultCursor('none')`;
- 创建 Graphics 对象绘制准星, 每帧根据 `input.mousePointer` 更新位置;
- 准星层级高于所有游戏元素 (敌人 / 飞剑 / 粒子), 永远在最上层;

## 10. 手感预期与验收标准

### 10.1 玩起来应该是什么感觉

**关键词**: 凝重, 写意, 张弛, 有灵性;

**一次完整发射的体验**:

1. 玩家观察战场, 决定打哪个敌人;
2. 移动鼠标对准目标, 悬浮剑随鼠标转向 — 玩家看到剑就知道会朝哪飞;
3. 按下空格, 角色身上短促白光 → 剑"嗖"地一声脱手, 笔直冲向目标;
4. 拖着流动的剑光痕和飘散的光屑, 飞剑穿过路径上 2~3 个敌人, 速度逐渐衰减;
5. 剑到达远端, 凝在空中那一瞬间, 拖尾收缩, 剑身微颤;
6. 剑开始回程, 玩家此时已经在走位 (因为没武器要躲), 剑画出一道柔和弧线追来;
7. 回程加速, 拖尾重新变长, 切过角色身边的敌人 (高伤害);
8. 剑接触角色, 一道白光闪过, 入鞘, 悬浮剑重新出现, 准星外圈 8 点重聚;
9. 玩家立刻可以再次发射, 但他可能会先走位几步, 再瞄准下一个目标;

整个循环约 1.5~2.5 秒, **每一秒玩家都在做决策**: 瞄准谁 / 怎么走位 / 何时按空格;

### 10.2 验收标准

实测时, 用以下问题验证"对不对":

**机制验收**:

- [ ] 同时只能存在一把飞剑;
- [ ] 池空时按空格无任何反应 (不积累, 不延迟);
- [ ] 去程是直线, 不偏转;
- [ ] 飞剑穿过敌人不停止, 减速;
- [ ] 飞剑到达 600px (D_max) 时开始转折;
- [ ] 回程追踪角色当前位置, 不是发射时的位置;
- [ ] 飞剑接触角色身体 = 入鞘, 立刻可以再发;
- [ ] 不管角色怎么跑, 飞剑总能追上;

**手感验收**:

- [ ] 飞剑出鞘"嗖"地一声有爆发感 (1000 px/s 够快);
- [ ] 单次穿透 3 个以上敌人时, 后面的敌人明显吃低伤害;
- [ ] 远端那一瞬间能感受到"凝住"的视觉静止;
- [ ] 回程曲线是柔和的, 不机械;
- [ ] 玩家有意识地走位时, 能感觉到走位影响了回程伤害分布;
- [ ] 池空的 1~2 秒里玩家会本能地走位躲避;

**视觉验收**:

- [ ] 飞剑能看出是"光中有剑", 不是纯光也不是纯剑;
- [ ] 拖尾长度随速度变化, 速度感强;
- [ ] 出鞘 / 命中 / 入鞘的白光闪烁清晰可见但不刺眼;
- [ ] 悬浮剑的流动呼吸明显但不抢戏;
- [ ] 悬浮剑指向鼠标的反应即时, 不滞后;
- [ ] 准星 8 点的有剑/无剑切换清晰;

**整体验收 (最重要)**:

- [ ] 玩家玩 5 分钟后, **能不看 HUD 就知道自己有没有剑**;
- [ ] 玩家玩 10 分钟后, **会主动用走位调整回程命中目标**;
- [ ] 玩家发射飞剑的瞬间, **觉得"爽", 不觉得"软"**;
- [ ] 飞剑回到手上的瞬间, **觉得"接住了", 不觉得"消失了"**;
- [ ] 整体气质让人想到"御剑", 不是"扔回旋镖", 也不是"打弹幕";

### 10.3 不对劲的早期信号

如果实测时出现以下情况, 说明某处需要调:

| 症状               | 可能的问题                | 调参方向                     |
| ------------------ | ------------------------- | ---------------------------- |
| 飞剑"软", 没爽感   | 初速太低 / 拖尾太短       | V_max ↑ 至 1200, 拖尾系数 ↑  |
| 飞剑"飘", 没份量   | 拖尾太长 / 远端速度太高   | 拖尾上限 ↓, V_min ↓ 至 150   |
| 群伤太强, 玩家乱射 | 锋利度太高                | 锋利度 ↓ 至 0.5              |
| 单体伤害不够       | 锋利度太低 / 伤害系数太低 | 伤害系数 ↑ 至 0.15           |
| 远端凝滞不明显     | V_min 太高                | V_min ↓ 至 150               |
| 回程太机械         | 追踪角速度太高            | 角速度 ↓ 至 120°/s           |
| 回程追不上         | V_min < 角色速度          | 重新校准 V_min, 或角色速度 ↓ |
| 看不清飞剑在哪     | 主体太小 / 颜色对比度低   | 尺寸 ↑ 至 60×5, 或柔光晕加强 |
| 玩家分不清池状态   | 悬浮剑视觉太弱            | 悬浮剑 alpha ↑ / 流动对比 ↑  |

## 11. 扩展性与未来形态

### 11.1 资源池容量成长 (M5+)

角色升级 → 飞剑容量上限 +1;

- 等级 X: 容量 = 1 (M2);
- 等级 Y: 容量 = 2;
- 等级 Z: 容量 = 3;
- ...

视觉表现: 悬浮剑数量同步增加, 以角色为中心呈环绕排列 (品字形 / 等分圆);

### 11.2 主动召回 (M5+ 升级技能)

- 触发: 玩家按下另一个键 (例如鼠标右键 / Shift), 主动召回外面所有飞剑;
- 行为: 飞剑立即进入回程状态, 不等到达最大距离;
- 效果: 缩短"空窗期", 玩家可以主动控制飞剑节奏;
- 叙事: "御剑修为提升, 可呼剑归来";

### 11.3 多种飞剑武器 (M5+)

差异化主要通过以下属性:

| 维度     | 示例                                         |
| -------- | -------------------------------------------- |
| 锋利度   | "斩魂剑" (0.9, 群伤) vs "重玄剑" (0.4, 单体) |
| 初速     | 高速短剑 vs 慢速重剑                         |
| 最大距离 | 短程剑 vs 长程剑                             |
| 颜色     | 青 / 紫 / 金 / 五色                          |
| 特殊效果 | 命中点燃 / 命中冰冻 / 命中分裂               |
| 自转     | 高级剑加入自转动画                           |

### 11.4 剑诀技能 (M5+)

消耗多把飞剑触发特殊效果, 类似法术系统:

| 技能     | 消耗 | 效果                               |
| -------- | ---- | ---------------------------------- |
| 万剑诀   | 全部 | 同时发射所有飞剑, 扇形覆盖         |
| 御剑斩   | 2    | 两把剑交叉飞行, 命中点产生剑气爆裂 |
| 凌空一剑 | 1    | 飞剑朝鼠标位置瞬移, 范围伤害       |
| ...      | ...  | ...                                |

### 11.5 近身攻击 (M3+)

当飞剑在外飞行 (池空) 时, 玩家可以发动近身攻击:

- 伤害低于飞剑;
- 范围小;
- 作为"空窗期"的补位手段, 不取代飞剑;
- 设计意图: 让"无剑"状态不至于完全无攻击能力, 但仍保留张力;

### 11.6 角色御剑能力成长 (M5+)

随角色等级, 御剑能力倍率提升:

- 初速倍率: 1.0 → 1.2 → 1.5 ...;
- 最大距离倍率: 1.0 → 1.1 → 1.3 ...;
- 锋利度倍率: 1.0 → 1.05 → 1.1 ...;
- 等等;

通过 `PlayerStats` 系统实现, M2 框架已经留好接口;

## 12. 实现路径建议

### 12.1 M2 实现顺序 (从底层到表面)

1. **物理与机制层** (无视觉)
   - 飞剑实体类 + 状态机 (OUTBOUND / TURNING / RETURNING);
   - 基于距离的速度衰减;
   - 笔直去程 + 中等追踪回程;
   - 穿透检测 + 锋利度减速;
   - 速度 × 伤害系数的伤害计算;
   - 资源池 + 空池行为;
   - 用占位 (例如纯色矩形) 显示飞剑, 先把机制跑通;

2. **属性系统抽象层**
   - `SwordConfig` 接口;
   - `PlayerStats` 御剑能力倍率;
   - 实例化时合并计算;
   - M2 配置为最简形态;

3. **视觉层 - 飞剑本体**
   - 梭形 + 柔光晕;
   - 关键瞬间反馈 (出鞘 / 命中 / 远端微颤 / 入鞘);

4. **视觉层 - 拖尾系统**
   - 渐变剑光痕 (长度耦合速度);
   - 点缀粒子;
   - 路径残痕;

5. **视觉层 - 悬浮剑**
   - 位置 (跟随光标转向);
   - 流动呼吸 (Tween 链);
   - 消失 / 出现 动画;

6. **UI 层**
   - 自定义准星;
   - 有剑 / 无剑两态切换;

### 12.2 调试与调参

- 建议加一个调试模式 (按 ` 键开启), 显示:
  - 飞剑当前速度数值;
  - 飞剑当前状态 (去程/转折/回程);
  - 飞剑距角色距离;
  - 池容量状态;
  - 命中伤害浮动数字;
- 所有数值参数集中在 `Config.ts`, 修改即生效, 不要硬编码;
- 实测建议: 单独写一个 "Dojo" 场景, 只有 1 个角色 + 静止假人 + 慢速移动假人 + 编队假人, 反复测手感;

### 12.3 风险提示

- **追踪强度调参是手感最敏感的部分**, 计划至少 2~3 轮迭代;
- **拖尾性能**: 如果用 Graphics 每帧重绘, 多剑时可能卡; M2 单剑没问题, M5+ 多剑要换 Rope 或 shader;
- **悬浮剑指向滞后**: 鼠标输入到悬浮剑转向之间不能有 lag, 必须在同一帧内完成;
- **入鞘判定**: "飞剑接触角色" 用碰撞圆判定, 半径建议 = 角色半径 + 5px, 避免擦边不算;
